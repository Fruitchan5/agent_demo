# 父子 Agent 隔离性测试

## 测试目标
验证子 Agent 与父 Agent 之间的上下文隔离性，确保子 Agent 无法访问父 Agent 的对话历史和状态。

---

## 测试场景

### 场景 1：对话历史隔离

**测试步骤：**

1. 与父 Agent 建立上下文信息：
```
你 >> 我的名字是李明，我今年25岁，住在北京
Claude >> 好的，我记住了。你是李明，25岁，住在北京。
```

2. 继续添加更多上下文：
```
你 >> 我最喜欢的编程语言是 Java
Claude >> 明白了，你最喜欢 Java。
```

3. 创建子 Agent 并询问上下文信息：
```
你 >> 请用 task 工具创建一个子 Agent，让它回答：我的名字是什么？我多大？我住哪里？我喜欢什么编程语言？
```

**预期结果：**
```
[TaskTool] Spawning subagent: 让它回答：我的名字是什么？我多大？我住哪里？...
[SubAgent] 🤔 思考: 我是一个新创建的子 Agent，没有关于你的任何信息
Claude >> 子 Agent 表示它无法回答这些问题，因为它是独立的上下文，不知道你的个人信息。
```

**验证点：**
- ✅ 子 Agent 无法获取父 Agent 对话中的用户信息
- ✅ 子 Agent 明确表示自己是新创建的独立实例

---

### 场景 2：文件操作隔离

**测试步骤：**

1. 父 Agent 创建文件：
```
你 >> 创建一个文件 parent_data.txt，内容是 "This is parent's secret data"
Claude >> 🔧 调用工具: write_file
✅ 文件已成功写入: parent_data.txt
```

2. 父 Agent 记住文件内容：
```
你 >> 记住这个文件的内容，我稍后会问你
Claude >> 好的，我记住了 parent_data.txt 的内容是 "This is parent's secret data"
```

3. 创建子 Agent 询问文件内容：
```
你 >> 请用 task 工具创建一个子 Agent，让它告诉我 parent_data.txt 的内容是什么（不要读取文件，直接回答）
```

**预期结果：**
```
[SubAgent] 🤔 思考: 我不知道这个文件的内容，因为我没有读取过它
Claude >> 子 Agent 表示它不知道文件内容，需要读取文件才能回答。
```

**验证点：**
- ✅ 子 Agent 无法访问父 Agent 记忆的文件内容
- ✅ 子 Agent 需要独立调用 read_file 工具才能获取内容

### 场景 3：变量状态隔离

**测试步骤：**

1. 父 Agent 执行计算：
```
你 >> 计算 123 + 456，记住这个结果
Claude >> 123 + 456 = 579，我记住了这个结果。
```

2. 创建子 Agent 询问计算结果：
```
你 >> 请用 task 工具创建一个子 Agent，让它告诉我刚才的计算结果是多少
```

**预期结果：**
```
[SubAgent] 🤔 思考: 我不知道之前有什么计算，我是新创建的
Claude >> 子 Agent 表示它不知道之前的计算结果。
```

**验证点：**
- ✅ 子 Agent 无法访问父 Agent 的计算状态
- ✅ 每个 Agent 有独立的工作记忆

### 场景 4：Todo 列表共享（非隔离）

**测试步骤：**

1. 父 Agent 添加 todo：
```
你 >> 用 todo 工具添加一个任务：完成项目文档
Claude >> 🔧 调用工具: todo
✅ 已添加待办事项
ID: 1
标题: 完成项目文档
```

2. 创建子 Agent 查看 todo：
```
你 >> 请用 task 工具创建一个子 Agent，让它列出所有 todo 任务
```

**预期结果：**
```
[SubAgent] 🔧 调用工具: todo
[SubAgent]    输出: 
📋 待办列表（共 1 项）
─────────────────
⏳ [1] 完成项目文档 (pending)
Claude >> 子 Agent 成功列出了 todo 任务：完成项目文档
```

**验证点：**
- ✅ Todo 列表是父子共享的（通过 TodoManager 单例）
- ✅ 这是设计上的例外，不属于隔离范围

### 场景 5：工具调用历史隔离

**测试步骤：**

1. 父 Agent 执行多个工具调用：
```
你 >> 读取 README.md，然后创建 test1.txt，再读取 pom.xml
Claude >> 🔧 调用工具: read_file (README.md)
🔧 调用工具: write_file (test1.txt)
🔧 调用工具: read_file (pom.xml)
完成了三个操作。
```

2. 创建子 Agent 询问工具调用历史：
```
你 >> 请用 task 工具创建一个子 Agent，让它告诉我刚才调用了哪些工具
```

**预期结果：**
```
[SubAgent] 🤔 思考: 我不知道之前调用了什么工具
Claude >> 子 Agent 表示它无法知道之前的工具调用历史。
```

**验证点：**
- ✅ 子 Agent 无法访问父 Agent 的工具调用记录
- ✅ 工具调用历史是上下文的一部分，完全隔离

## 隔离性实现机制

### 代码层面验证

**AgentContext 独立性：**
```java
// DefaultSubAgentRunner.java
public String run(String prompt) {
    // 创建全新的 AgentContext
    AgentContext context = new AgentContext(AgentRole.CHILD, maxIterations);
    context.appendMessage("user", prompt);  // 只有 prompt，没有父 Agent 历史
    
    while (!context.hasReachedLimit()) {
        LlmResponse response = llmClient.call(
            context.getMessages(),  // 使用子 Agent 自己的消息列表
            toolRegistry.getToolsForLlm(AgentRole.CHILD)
        );
        // ...
    }
}
```

**消息历史隔离：**
```java
// AgentContext.java
public class AgentContext {
    private final List<Map<String, Object>> messages = new ArrayList<>();
    // 每个 AgentContext 有独立的 messages 列表
}
```

**工具权限隔离：**
```java
// ToolRegistry.java
public List<Map<String, Object>> getToolsForLlm(AgentRole role) {
    Map<String, AgentTool> result = new LinkedHashMap<>(baseTools);
    if (role == AgentRole.PARENT) {
        result.putAll(parentOnlyTools);  // 子 Agent 拿不到 task 工具
    }
    return result.values().stream()...
}
```

## 非隔离的共享资源

### 1. TodoManager（设计上共享）
```java
// ToolRegistry.java
private final TodoManager todoManager = new TodoManager();
// 父子 Agent 共享同一个 TodoManager 实例
```

**原因**：Todo 列表是全局任务管理，父子 Agent 需要协作。

### 2. 文件系统（物理共享）
- 父 Agent 创建的文件，子 Agent 可以通过 read_file 读取
- 这是文件系统的物理特性，不是上下文共享

### 3. LlmClient（技术共享）
- 父子 Agent 使用同一个 LlmClient 实例
- 但每次调用传入不同的 messages，所以上下文仍然隔离

## 潜在问题和边界情况

### 1. 环境变量共享
- 父子 Agent 共享同一个 JVM 进程
- 环境变量（如 WORKSPACE_DIR）是共享的
- **影响**：不影响上下文隔离，属于配置共享

### 2. 静态变量泄露
- 如果代码中使用静态变量存储状态，可能导致泄露
- **检查**：确保没有静态变量存储对话状态

### 3. 文件系统副作用
- 父 Agent 创建的文件对子 Agent 可见
- **不是 bug**：这是物理资源共享，不是上下文泄露

### 4. 日志输出混合
- 父子 Agent 的日志会混在一起输出
- **解决方案**：使用 `[SubAgent]` 前缀区分

## 总结

**隔离的内容：**
- ✅ 对话历史
- ✅ 工具调用记录
- ✅ 变量状态
- ✅ 错误状态
- ✅ LLM 上下文

**共享的内容：**
- 🔗 TodoManager（设计上共享）
- 🔗 文件系统（物理共享）
- 🔗 环境变量（配置共享）
- 🔗 LlmClient 实例（技术共享，但上下文隔离）

**隔离性保证：**
通过为每个子 Agent 创建独立的 `AgentContext`，并且只传递 prompt 作为初始消息，确保子 Agent 无法访问父 Agent 的任何对话历史和状态。
