# Agent Demo 项目测试范围

## 项目概述
Java 实现的 AI Agent 系统，模仿 Claude API 的工具调用机制，支持父子 Agent 架构。

---

## 测试层级划分

### 1. 单元测试（Unit Tests）
**范围**：独立测试各个类和方法

#### 1.1 工具类测试
- **WorkspaceEnv**
  - ✅ 路径安全检查（防止路径遍历攻击）
  - ✅ 相对路径解析
  - ✅ 工作目录限制
  - ❌ 非法路径拒绝（`../`, 绝对路径等）

- **BashTool**
  - ✅ 命令执行成功
  - ✅ 命令执行失败处理
  - ✅ 输出捕获
  - ❌ 危险命令拦截（可选）

- **ReadFileTool**
  - ✅ 读取存在的文件
  - ❌ 读取不存在的文件
  - ❌ 读取无权限的文件
  - ✅ 大文件处理

- **WriteFileTool**
  - ✅ 写入新文件
  - ✅ 覆盖已存在文件
  - ❌ 写入受保护目录
  - ✅ 创建父目录

- **EditFileTool**
  - ✅ 替换文件内容
  - ❌ 替换不存在的内容
  - ✅ 多次匹配只替换第一次
  - ❌ 文件不存在时的处理

- **TodoTool**
  - ✅ 添加待办事项
  - ✅ 更新待办状态
  - ✅ 列出所有待办
  - ✅ 删除待办事项
  - ❌ 无效操作处理

- **TaskTool**
  - ✅ 创建子 Agent
  - ✅ 传递 prompt 参数
  - ✅ 接收子 Agent 结果

#### 1.2 核心类测试
- **LlmClient**
  - ✅ API 调用成功
  - ❌ API 调用失败（网络错误、认证失败）
  - ✅ JSON 序列化/反序列化
  - ❌ 超时处理

- **ToolRegistry**
  - ✅ 注册基础工具
  - ✅ 注册父端专属工具
  - ✅ 根据角色获取工具列表
  - ✅ 工具名称查找

- **TodoManager**
  - ✅ CRUD 操作
  - ✅ ID 自增
  - ✅ 状态转换
  - ❌ 并发访问（如果需要）

- **DefaultSubAgentRunner**
  - ✅ 子 Agent 执行成功
  - ✅ 迭代限制生效
  - ❌ 子 Agent 执行异常
  - ✅ 结果摘要生成

#### 1.3 POJO 测试
- **ContentBlock**
  - ✅ 不同类型的序列化/反序列化
  - ✅ 字段完整性

- **LlmResponse**
  - ✅ stopReason 解析
  - ✅ content 列表处理

### 5. 安全测试（Security Tests）
**范围**：安全漏洞和攻击防护

#### 5.1 路径遍历攻击
- 🔒 `../../../etc/passwd`
- 🔒 绝对路径 `/etc/passwd`
- 🔒 符号链接跳出工作目录

#### 5.2 命令注入
- 🔒 Bash 命令中的特殊字符（`;`, `|`, `&&`）
- 🔒 恶意脚本执行

#### 5.3 API 密钥泄露
- 🔒 日志中不输出 API Key
- 🔒 错误消息不包含敏感信息
