# S09 设计文档：Agent Teams

## 概述

s09 实现持久化的 Agent 团队协作机制，允许 lead agent 创建多个 teammate，通过 JSONL 文件进行异步通信。

## 核心架构

```
TeammateManager
  ├─ ExecutorService (CachedThreadPool, daemon threads)
  ├─ Map<String, TeammateSession> sessions
  ├─ TeamConfig config (.team/config.json)
  └─ MessageBus messageBus

MessageBus
  ├─ Path inboxDir (.team/inbox/)
  ├─ send(from, to, content, type)
  ├─ readInbox(name) -> List<Message>  // 读取并清空
  └─ broadcast(from, content, teammates)

TeammateLoop (Runnable)
  ├─ 一次性执行（最多 50 轮）
  ├─ 每轮开始检查 inbox
  ├─ 调用 LLM + 执行工具
  └─ 完成后状态 -> IDLE
```

## 数据结构

### .team/config.json
```json
{
  "teamName": "default",
  "members": [
    {
      "name": "alice",
      "role": "backend-dev",
      "status": "WORKING",
      "spawnedAt": "2026-05-01T12:00:00Z"
    }
  ]
}
```

### .team/inbox/alice.jsonl
```json
{"type":"MESSAGE","from":"lead","content":"实现登录接口","timestamp":1714567890123}
{"type":"BROADCAST","from":"lead","content":"phase 1 complete","timestamp":1714567900456}
```

### Message 类
```java
class Message {
    String type;       // MESSAGE, BROADCAST
    String from;       // 发送者名称
    String content;    // 消息内容
    long timestamp;    // 时间戳（毫秒）
}
```

## 工具定义

### Lead 工具（9 个）
1. **spawn_teammate** (parentOnly)
   - 创建持久化队友，在独立线程中运行
   - 参数：name, role, prompt
   
2. **list_teammates** (parentOnly)
   - 列出所有队友及其状态
   
3. **broadcast** (parentOnly)
   - 向所有队友广播消息
   - 参数：content

4. **send_message** (baseTools)
   - 向指定队友发送消息
   - 参数：to, content, msg_type (可选)

5. **read_inbox** (baseTools)
   - 读取并清空自己的收件箱

6-9. bash, read_file, write_file, edit_file (baseTools)

### Teammate 工具（6 个）
- bash, read_file, write_file, edit_file
- send_message, read_inbox

**不包含**：spawn_teammate, list_teammates, broadcast, task, compact

## 生命周期

```
spawn -> WORKING (线程启动) -> 执行 50 轮或完成 -> IDLE (线程结束)
       ↑                                                    |
       +-------- 重新 spawn (创建新线程) -------------------+
```

- **WORKING**：线程正在执行
- **IDLE**：线程已结束，任务完成
- **重新激活**：对 IDLE 的 teammate 调用 spawn，会创建新线程重新执行

## 通信机制

### 发送消息
```java
messageBus.send("lead", "alice", "fix bug", "MESSAGE");
// 追加到 .team/inbox/alice.jsonl
```

### 读取消息
```java
List<Message> inbox = messageBus.readInbox("alice");
// 读取 .team/inbox/alice.jsonl 的所有行
// 清空文件（drain）
```

### 广播
```java
messageBus.broadcast("lead", "status update", List.of("alice", "bob"));
// 向 alice 和 bob 发送 BROADCAST 类型消息
```

## 线程模型

使用 `Executors.newCachedThreadPool()` 管理线程：
- 动态扩展（无上限）
- 空闲 60 秒后回收
- daemon 线程（不阻止 JVM 退出）

每次 spawn 提交一个 Runnable 到线程池，执行完自动结束。

## AgentRole 扩展

```java
public enum AgentRole {
    PARENT,    // lead agent
    CHILD,     // 一次性 subagent (task 工具)
    TEAMMATE   // 持久化 teammate (新增)
}
```

## 与现有组件的集成

### ToolRegistry
- `spawn_teammate`, `list_teammates`, `broadcast` 注册到 `parentOnlyTools`
- `send_message`, `read_inbox` 注册到 `baseTools`
- `getToolsForLlm(AgentRole.TEAMMATE)` 返回 6 个工具

### AgentLoop
- 在 `runAgentCycle()` 开始时自动检查 lead 的 inbox
- 如果有消息，注入到 chatHistory：
  ```java
  List<Message> inbox = messageBus.readInbox("lead");
  if (!inbox.isEmpty()) {
      String content = "<inbox>" + toJson(inbox) + "</inbox>";
      chatHistory.add(Map.of("role", "user", "content", content));
  }
  ```

## 配置项

在 `.env` 中添加：
```
TEAMMATE_MAX_ITERATIONS=50
```

## 测试场景

1. **基本通信**
   - spawn alice (coder)
   - send_message to alice: "implement login"
   - alice 执行任务，完成后发送结果给 lead
   - lead 通过 read_inbox 查看回复

2. **广播**
   - spawn alice, bob
   - broadcast "phase 1 complete"
   - alice 和 bob 都收到消息

3. **重新激活**
   - spawn alice，执行完进入 IDLE
   - 再次 spawn alice，创建新线程重新执行

## 相对 s08 的变更

| 组件       | s08                | s09                        |
|------------|--------------------|-----------------------------|
| Tools      | 12                 | 17 (+5: spawn/list/broadcast/send/read) |
| Agent 数量 | 单一               | lead + N 个 teammates       |
| 持久化     | 无                 | .team/config.json + JSONL inbox |
| 线程       | BackgroundTaskManager (shell only) | CachedThreadPool (agent loop) |
| 生命周期   | 一次性             | spawn -> work -> idle -> respawn |
| 通信       | 无                 | message + broadcast         |
