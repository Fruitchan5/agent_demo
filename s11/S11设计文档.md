# S11 自主代理 - Java 实现设计文档

## 一、核心概念

S11 在 s09/s10 基础上实现**自主性（Autonomy）**：
- **s09**: 被动工作模式（spawn → work → idle → 线程结束）
- **s10**: 添加协议系统（shutdown_request, plan_approval）
- **s11**: 自主工作模式（idle 时主动轮询任务板，自动认领任务）

**核心洞察**："Agent 自己找工作"

## 二、功能模块划分

### 模块 1：任务板系统（Task Board）
**职责**：管理共享任务队列，支持任务扫描和认领

**核心类**：
```java
// 1.1 任务实体
public class Task {
    private int id;
    private String subject;
    private String description;
    private TaskStatus status;  // PENDING, IN_PROGRESS, COMPLETED
    private String owner;       // null 表示未认领
    private List<Integer> blockedBy;
    private Instant createdAt;
    private Instant updatedAt;
}

// 1.2 任务板管理器
public class TaskBoard {
    private final Path tasksDir;  // .tasks/
    private final Lock claimLock; // 防止并发认领冲突
    
    // 扫描未认领任务（无锁读取）
    List<Task> scanUnclaimedTasks();
    
    // 原子性认领任务（加锁写入）
    ClaimResult claimTask(int taskId, String owner);
    
    // 更新任务状态
    void updateTaskStatus(int taskId, TaskStatus status);
    
    // 加载/保存任务
    Task loadTask(int taskId);
    void saveTask(Task task);
}

// 1.3 认领结果
public class ClaimResult {
    private boolean success;
    private String message;
    private Task task;  // 成功时返回任务详情
}
```

**设计要点**：
- 使用 `ReentrantLock` 保护认领操作，防止双重认领
- 扫描操作无锁（只读），认领操作加锁（写入）
- 任务文件格式：`.tasks/task_{id}.json`
- 认领条件：`status == PENDING && owner == null && blockedBy.isEmpty()`

### 模块 2：空闲轮询循环（Idle Polling Loop）
**职责**：teammate 完成工作后进入 IDLE 状态，定期轮询收件箱和任务板

**核心类**：
```java
public class IdlePoller {
    private static final int POLL_INTERVAL_SECONDS = 5;
    private static final int IDLE_TIMEOUT_SECONDS = 60;
    
    private final String teammateName;
    private final MessageBus messageBus;
    private final TaskBoard taskBoard;
    
    // 执行空闲轮询，返回是否找到新工作
    public IdlePollResult poll() {
        int maxPolls = IDLE_TIMEOUT_SECONDS / POLL_INTERVAL_SECONDS;
        
        for (int i = 0; i < maxPolls; i++) {
            // 1. 检查收件箱
            List<Message> inbox = messageBus.readInbox(teammateName);
            if (!inbox.isEmpty()) {
                return IdlePollResult.foundMessages(inbox);
            }
            
            // 2. 扫描任务板
            List<Task> unclaimed = taskBoard.scanUnclaimedTasks();
            if (!unclaimed.isEmpty()) {
                Task task = unclaimed.get(0);
                ClaimResult result = taskBoard.claimTask(task.getId(), teammateName);
                if (result.isSuccess()) {
                    return IdlePollResult.claimedTask(result.getTask());
                }
                // 认领失败（被其他 agent 抢先），继续轮询
            }
            
            // 3. 等待下一轮
            Thread.sleep(POLL_INTERVAL_SECONDS * 1000);
        }
        
        return IdlePollResult.timeout();
    }
}

// 轮询结果
public class IdlePollResult {
    private IdlePollType type;  // MESSAGES, TASK_CLAIMED, TIMEOUT
    private List<Message> messages;
    private Task claimedTask;
    
    public static IdlePollResult foundMessages(List<Message> msgs);
    public static IdlePollResult claimedTask(Task task);
    public static IdlePollResult timeout();
}
```

**设计要点**：
- 轮询顺序：先检查收件箱（响应式），再扫描任务板（主动式）
- 使用 `Thread.sleep()` 实现轮询间隔
- 认领失败时继续轮询（乐观并发控制）
- 超时后返回 TIMEOUT，触发线程关闭

### 模块 3：身份重注入（Identity Re-injection）
**职责**：上下文压缩后恢复 agent 身份信息

**核心类**：
```java
public class IdentityManager {
    
    // 创建身份块
    public static Map<String, Object> makeIdentityBlock(
        String name, 
        String role, 
        String teamName
    ) {
        String content = String.format(
            "<identity>You are '%s', role: %s, team: %s. Continue your work.</identity>",
            name, role, teamName
        );
        return Map.of("role", "user", "content", content);
    }
    
    // 判断是否需要重注入（消息列表过短，表明发生了压缩）
    public static boolean needsReinjection(List<Map<String, Object>> messages) {
        return messages.size() <= 3;
    }
    
    // 重注入身份
    public static void reinjectIdentity(
        List<Map<String, Object>> messages,
        String name,
        String role,
        String teamName
    ) {
        if (needsReinjection(messages)) {
            messages.add(0, makeIdentityBlock(name, role, teamName));
            messages.add(1, Map.of(
                "role", "assistant",
                "content", "I am " + name + ". Continuing."
            ));
        }
    }
}
```

**设计要点**：
- 触发条件：`messages.size() <= 3`（表明发生了上下文压缩）
- 注入位置：在消息列表开头插入身份块 + 确认消息
- 与现有的 `ContextCompactor` 集成

### 模块 4：增强的 TeammateLoop
**职责**：扩展现有的 `TeammateManager.teammateLoop()`，添加 IDLE 阶段

**修改点**：
```java
public class TeammateManager {
    
    private void teammateLoop(String name, String role, String prompt) {
        String teamName = config.getTeamName();
        String systemPrompt = String.format(
            "You are '%s', role: %s, team: %s, at %s. " +
            "Use idle tool when you have no more work. You will auto-claim new tasks.",
            name, role, teamName, System.getProperty("user.dir")
        );
        
        List<Map<String, Object>> messages = new ArrayList<>();
        messages.add(Map.of("role", "user", "content", prompt));
        
        List<Map<String, Object>> tools = toolRegistry.getToolsForLlm(AgentRole.TEAMMATE);
        
        IdlePoller poller = new IdlePoller(name, messageBus, taskBoard);
        
        while (true) {
            // ===== WORK 阶段 =====
            boolean idleRequested = workPhase(name, messages, tools, systemPrompt);
            
            if (!idleRequested) {
                // 非正常结束（异常或达到最大迭代次数）
                break;
            }
            
            // ===== IDLE 阶段 =====
            updateMemberStatus(name, "IDLE");
            IdlePollResult pollResult = poller.poll();
            
            switch (pollResult.getType()) {
                case MESSAGES:
                    // 收到消息，恢复工作
                    for (Message msg : pollResult.getMessages()) {
                        messages.add(Map.of("role", "user", "content", msg.toJson()));
                    }
                    updateMemberStatus(name, "WORKING");
                    continue;
                    
                case TASK_CLAIMED:
                    // 认领到任务，恢复工作
                    Task task = pollResult.getClaimedTask();
                    
                    // 检查是否需要重注入身份
                    IdentityManager.reinjectIdentity(messages, name, role, teamName);
                    
                    // 添加任务提示
                    String taskPrompt = String.format(
                        "<auto-claimed>Task #%d: %s\n%s</auto-claimed>",
                        task.getId(), task.getSubject(), task.getDescription()
                    );
                    messages.add(Map.of("role", "user", "content", taskPrompt));
                    messages.add(Map.of(
                        "role", "assistant",
                        "content", "Claimed task #" + task.getId() + ". Working on it."
                    ));
                    
                    updateMemberStatus(name, "WORKING");
                    continue;
                    
                case TIMEOUT:
                    // 超时，关闭线程
                    updateMemberStatus(name, "SHUTDOWN");
                    return;
            }
        }
    }
    
    // WORK 阶段逻辑（提取为独立方法）
    private boolean workPhase(
        String name,
        List<Map<String, Object>> messages,
        List<Map<String, Object>> tools,
        String systemPrompt
    ) {
        for (int i = 0; i < MAX_ITERATIONS; i++) {
            // 检查收件箱（每轮开始）
            List<Message> inbox = messageBus.readInbox(name);
            for (Message msg : inbox) {
                if ("SHUTDOWN_REQUEST:v1".equals(msg.getType())) {
                    updateMemberStatus(name, "SHUTDOWN");
                    return false;  // 立即退出
                }
                messages.add(Map.of("role", "user", "content", msg.toJson()));
            }
            
            // 调用 LLM
            LlmResponse response = llmClient.call(messages, tools, systemPrompt);
            messages.add(Map.of("role", "assistant", "content", response.getContent()));
            
            if (!"tool_use".equals(response.getStopReason())) {
                return false;  // 非工具调用结束，退出
            }
            
            // 执行工具
            List<Map<String, Object>> toolResults = new ArrayList<>();
            boolean idleRequested = false;
            
            for (ContentBlock block : response.getContent()) {
                if ("tool_use".equals(block.getType())) {
                    if ("idle".equals(block.getName())) {
                        idleRequested = true;
                        toolResults.add(Map.of(
                            "type", "tool_result",
                            "tool_use_id", block.getId(),
                            "content", "Entering idle phase. Will poll for new tasks."
                        ));
                    } else {
                        String output = executeTeammateTool(name, block.getName(), block.getInput());
                        toolResults.add(Map.of(
                            "type", "tool_result",
                            "tool_use_id", block.getId(),
                            "content", output
                        ));
                    }
                }
            }
            
            messages.add(Map.of("role", "user", "content", toolResults));
            
            if (idleRequested) {
                return true;  // 进入 IDLE 阶段
            }
        }
        
        return false;  // 达到最大迭代次数
    }
}
```

## 三、Java 特性利用

### 3.1 并发控制
```java
// 使用 ReentrantLock 保护任务认领
public class TaskBoard {
    private final ReentrantLock claimLock = new ReentrantLock();
    
    public ClaimResult claimTask(int taskId, String owner) {
        claimLock.lock();
        try {
            Task task = loadTask(taskId);
            
            // 验证认领条件
            if (task.getOwner() != null) {
                return ClaimResult.error("Task already claimed by " + task.getOwner());
            }
            if (task.getStatus() != TaskStatus.PENDING) {
                return ClaimResult.error("Task status is " + task.getStatus());
            }
            if (!task.getBlockedBy().isEmpty()) {
                return ClaimResult.error("Task is blocked");
            }
            
            // 原子性更新
            task.setOwner(owner);
            task.setStatus(TaskStatus.IN_PROGRESS);
            task.setUpdatedAt(Instant.now());
            saveTask(task);
            
            return ClaimResult.success(task);
        } finally {
            claimLock.unlock();
        }
    }
}
```

### 3.4 Stream API
```java
public class TaskBoard {
    public List<Task> scanUnclaimedTasks() {
        try {
            return Files.list(tasksDir)
                .filter(p -> p.getFileName().toString().startsWith("task_"))
                .map(this::loadTaskSafe)
                .filter(Optional::isPresent)
                .map(Optional::get)
                .filter(t -> t.getStatus() == TaskStatus.PENDING)
                .filter(t -> t.getOwner() == null)
                .filter(t -> t.getBlockedBy().isEmpty())
                .sorted(Comparator.comparing(Task::getId))
                .collect(Collectors.toList());
        } catch (IOException e) {
            return Collections.emptyList();
        }
    }
    
    private Optional<Task> loadTaskSafe(Path path) {
        try {
            return Optional.of(MAPPER.readValue(path.toFile(), Task.class));
        } catch (IOException e) {
            return Optional.empty();
        }
    }
}
```

## 八、潜在问题与解决方案

### 8.1 文件系统竞争
**问题**：多个 agent 同时读写任务文件

**解决**：
- 扫描操作无锁（只读，允许脏读）
- 认领操作加锁（写入，保证原子性）
- 使用 `ReentrantLock` 而非文件锁（性能更好）

### 8.2 轮询开销
**问题**：多个 agent 频繁轮询，API 调用成本高

**解决**：
- 合理设置轮询间隔（默认 5 秒）
- 空闲超时后关闭线程（默认 60 秒）
- 未来可优化为事件驱动（文件监听）

### 8.3 身份丢失
**问题**：上下文压缩后 agent 忘记自己是谁

**解决**：
- 检测消息列表长度（`<= 3` 表明压缩）
- 重新注入身份块到消息列表开头
- 包含 name, role, team 信息

### 8.4 任务认领冲突
**问题**：两个 agent 同时认领同一任务

**解决**：
- 使用 `ReentrantLock` 序列化认领操作
- 第二个 agent 收到错误，继续轮询下一个任务
- 乐观并发控制，失败后重试

## 十、总结

S11 的核心是**自主性**，通过以下机制实现：

1. **任务板**：共享工作队列，支持扫描和认领
2. **空闲轮询**：IDLE 阶段定期检查收件箱和任务板
3. **身份重注入**：压缩后恢复 agent 身份
4. **工具扩展**：`idle` 和 `claim_task` 工具
